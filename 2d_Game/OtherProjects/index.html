Project: 2d Bullet hell

Introducing 2D bullet hell game

This is a 2d bullet hell where players go through randomly generated dungeons and fight enemies. There are functional heal, damage system random loot drops, and a leveling system. The game was written in C++ and doesn't use a game engine Because of the simple physics I used SMFL and would draw, erease, and animate picutres to create a fun gameplay experience. This was a solo project.

I did not made the art however I did code all the systems mention above myself. All art was free assets found online credit given in video, and in the about section.

About Project name

This game is loosely based on Ream of the Mad God, which I enjoyed playing growing up. I created a unique algorithm for each type of dungeon to add to replay ability. The health and movement system are rather basic but still quite functional. The enemies spawn and despawn just outside of player vision to help game function smoothly.

Details about the code

All the players animation show up on the player picture and depending on which direction the player is moving a square is draw to only show to simulate animation. The map is generated using procedural generation as the game starts it spawns rooms in a direction then changes direction at a random size then spawns a boss room at the end. Then walls spawn after the map is generated in the respective spots.

Walls spawn after rooms to prevent inproper overlapping within the game.
